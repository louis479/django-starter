# Swahilipot Learning Platform

A Django-based learning platform prototype for managing courses, enrollments, and role-based access for students and instructors.

This project includes a refreshed Tailwind-powered interface, authentication flows, an instructor course-creation workflow, and a student enrollment flow. It is designed as a compact full-stack learning project that can run locally and can also be prepared for deployment with environment-based settings.

## Overview

The application supports two main user roles:

- `Student`: can register, log in, browse courses, and enroll in them.
- `Instructor`: can register, log in, and create courses.

After authentication, users land on a dashboard with role-aware navigation and quick actions. The UI has been updated to give the app a cleaner, more modern look across login, registration, dashboard, course listing, and course creation pages.

## Features

- User registration and login
- Role-based profiles for students and instructors
- Instructor-only course creation
- Student-only course enrollment
- Dashboard with role-aware navigation
- Tailwind-styled UI templates
- Static asset handling with WhiteNoise
- Deployment-oriented Django settings
- Basic automated tests for core auth and course flows

## Tech Stack

- Python
- Django 4.2
- SQLite for local development
- PostgreSQL via `DATABASE_URL` for Render and production
- Tailwind CSS via CDN
- WhiteNoise for static files
- Gunicorn for deployment

## Local Setup

1. Clone the repository.
2. Move into the Django project directory:

```bash
cd Learningplatform
```

3. Create and activate a virtual environment:

```bash
python3 -m venv ../.venv
source ../.venv/bin/activate
```

4. Install dependencies:

```bash
pip install -r requirements.txt
```

5. Run migrations:

```bash
python manage.py migrate
```

6. Start the development server:

```bash
python manage.py runserver
```

7. Open the app in your browser:

```text
http://127.0.0.1:8000/
```

## Environment Configuration

This project now supports environment-based configuration for deployment.

Use the included example file:

```bash
cp .env.example .env
```

Then update the values in `.env`, especially:

- `SECRET_KEY`
- `DEBUG`
- `ALLOWED_HOSTS`
- `CSRF_TRUSTED_ORIGINS`
- `DATABASE_URL` when using PostgreSQL in production

## Running Tests

The project includes basic tests for:

- registration
- login
- student enrollment
- instructor-only course creation access

Run them with:

```bash
python manage.py test accounts
```

## Static Files

Static files are configured for local development and deployment:

- `static/` contains source static assets
- `staticfiles/` is generated by `collectstatic`
- WhiteNoise is enabled to serve collected static assets in production

Before deployment, run:

```bash
python manage.py collectstatic --noinput
```

## Deployment Notes

The project includes:

- production-aware Django settings
- secure cookie and HTTPS-related settings
- a `Procfile` for Gunicorn-based deployment
- WhiteNoise for static asset serving
- automatic PostgreSQL support through `DATABASE_URL`

A production check can be run with:

```bash
python manage.py check --deploy
```

## Deploying on Render

This project is now set up to work well with a free Render Web Service plus a free Render Postgres database.

### 1. Create a Render Postgres database

In Render:

1. Create a new PostgreSQL database.
2. Once it is ready, copy its **Internal Database URL** or **External Database URL**.
3. Add that value to your web service as:

```text
DATABASE_URL=<your-render-postgres-url>
```

### 2. Create the Render Web Service

Use these values in the Render form:

- **Root Directory**: `Learningplatform`
- **Build Command**:

```bash
pip install -r requirements.txt && python manage.py collectstatic --noinput
```

- **Start Command**:

```bash
gunicorn Learningplatform.wsgi:application
```

### 3. Add Environment Variables

Add at least:

```text
SECRET_KEY=your-long-random-secret
DEBUG=False
ALLOWED_HOSTS=your-service-name.onrender.com
CSRF_TRUSTED_ORIGINS=https://your-service-name.onrender.com
DATABASE_URL=your-render-postgres-url
SECURE_SSL_REDIRECT=True
SESSION_COOKIE_SECURE=True
CSRF_COOKIE_SECURE=True
SECURE_HSTS_SECONDS=31536000
SECURE_HSTS_INCLUDE_SUBDOMAINS=True
SECURE_HSTS_PRELOAD=True
DEFAULT_FROM_EMAIL=noreply@your-service-name.onrender.com
```

### 4. Run Migrations

After the service is created, run:

```bash
python manage.py migrate
```

If Render shows a **Pre-Deploy Command** field, you can set:

```bash
python manage.py migrate
```
